# Use Case Diagram (текстовое представление)

## Акторы

* Ученик
* Учитель
* Родитель
* Сервис уведомлений (внешняя система)

---

## Use Cases ученика

### UC-01 Просмотреть список занятий

**Цель:** найти доступные занятия.

**Основной сценарий:**

1. Ученик открывает каталог занятий.
2. Система отображает список доступных занятий.
3. Ученик может использовать фильтры.

**Результат:**
Ученик видит актуальный список занятий.

---

### UC-02 Просмотреть карточку занятия

**Цель:** получить подробную информацию.

**Основной сценарий:**

1. Ученик выбирает занятие.
2. Система отображает карточку занятия.

**Результат:**
Ученик получает подробную информацию о занятии.

---

### UC-03 Записаться на занятие

**Цель:** стать участником занятия.

**Предусловия:**

* ученик авторизован;
* на занятии есть свободные места.

**Основной сценарий:**

1. Ученик открывает карточку занятия.
2. Нажимает «Записаться».
3. Система проверяет доступность мест.
4. Система создаёт запись.
5. Система отправляет уведомление.

**Альтернативный сценарий:**

* мест нет → отображается сообщение о недоступности записи.

**Результат:**
Ученик записан на занятие.

---

### UC-04 Отменить запись

**Предусловия:**

* ученик записан на занятие.

**Основной сценарий:**

1. Ученик открывает список своих занятий.
2. Выбирает занятие.
3. Нажимает «Отменить запись».
4. Система переводит запись в статус «отменена».

**Результат:**
Запись отменена.

---

### UC-05 Просмотреть мои занятия

**Цель:** увидеть все будущие записи.

**Основной сценарий:**

1. Ученик открывает раздел «Мои занятия».
2. Система отображает список записей.

**Результат:**
Ученик видит свои активные записи.

---

# Use Cases учителя

---

### UC-06 Создать занятие

**Цель:** добавить новое занятие.

**Основной сценарий:**

1. Учитель открывает раздел управления занятиями.
2. Нажимает «Создать занятие».
3. Заполняет:

   * название;
   * описание;
   * дату;
   * время;
   * лимит мест.
4. Сохраняет занятие.
5. Система создаёт запись.

**Результат:**
Новое занятие опубликовано.

---

### UC-07 Изменить занятие

**Предусловия:**

* занятие существует.

**Основной сценарий:**

1. Учитель открывает карточку занятия.
2. Вносит изменения.
3. Сохраняет изменения.
4. Система обновляет данные.
5. Система уведомляет участников.

**Результат:**
Информация о занятии обновлена.

---

### UC-08 Отменить занятие

**Предусловия:**

* занятие существует.

**Основной сценарий:**

1. Учитель открывает карточку занятия.
2. Нажимает «Отменить занятие».
3. Подтверждает действие.
4. Система меняет статус занятия.
5. Система отправляет уведомления участникам.

**Результат:**
Занятие отменено.

---

### UC-09 Просмотреть участников занятия

**Предусловия:**

* занятие существует.

**Основной сценарий:**

1. Учитель открывает карточку занятия.
2. Выбирает раздел «Участники».
3. Система отображает список участников.

**Результат:**
Учитель видит записавшихся учеников.

---

# Use Cases родителя

---

### UC-10 Получить уведомление

**Основной сценарий:**

1. Система фиксирует событие.
2. Формирует уведомление.
3. Отправляет уведомление родителю.

**События:**

* запись ребёнка;
* отмена занятия;
* изменение занятия.

**Результат:**
Родитель получает актуальную информацию.
