# Основные проблемы AS-IS

Именно их потом нужно будет вынести отдельным блоком под диаграммой.

### P1

Учитель вручную собирает участников.

### P2

Нет единого списка записавшихся.

### P3

Количество мест контролируется вручную.

### P4

Изменения могут потеряться в переписке.

### P5

Родитель зависит от того, передал ли ребёнок информацию.

# Mermaid-диаграммы процессов

## AS-IS

```mermaid
flowchart LR
    A[Учитель публикует сообщение в чате]
    B[Ученик читает сообщение]
    C{Ученик хочет участвовать?}
    D[Ученик пишет учителю]
    E[Учитель вручную добавляет ученика в список]
    F{Есть свободные места?}
    G[Учитель подтверждает запись]
    H[Ученик записан]
    I[Запись не выполнена]

    A --> B
    B --> C
    C -->|Да| D
    C -->|Нет| I
    D --> E
    E --> F
    F -->|Да| G
    G --> H
    F -->|Нет| I
````
