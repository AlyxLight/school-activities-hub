# API Contract MVP

## 1. Получить список занятий

```http
GET /activities
```

### Query-параметры

| Параметр  | Тип  | Обяз. | Описание               |
| --------- | ---- | ----: | ---------------------- |
| subjectId | UUID |   нет | Фильтр по предмету     |
| classId   | UUID |   нет | Фильтр по классу       |
| dateFrom  | date |   нет | Дата начала периода    |
| dateTo    | date |   нет | Дата окончания периода |

### Успешный ответ `200 OK`

```json
{
  "items": [
    {
      "id": "b7f3f6b2-2f43-4d8a-9c8f-9f2d1c1a1111",
      "title": "Консультация по ОГЭ",
      "subject": "Английский язык",
      "className": "9Б",
      "date": "2026-09-15",
      "startTime": "15:30",
      "endTime": "16:15",
      "availablePlaces": 4,
      "status": "published"
    }
  ]
}
```

  "startTime": "15:30",

  "status": "published"

| Код | Когда              |

## 3. Создать занятие

```http
POST /activities
```

### Request body

```json
{
  "title": "Консультация по ОГЭ",
  "description": "Разбор задания 35 письменной части",
  "subjectId": "uuid",
  "classId": "uuid",
  "date": "2026-09-15",
  "startTime": "15:30",
  "endTime": "16:15",
  "participantLimit": 15
}
```

### Успешный ответ `201 Created`

```json
{
  "id": "b7f3f6b2-2f43-4d8a-9c8f-9f2d1c1a1111",
  "status": "published"
}
```

### Ошибки

| Код | Когда                             |
| --- | --------------------------------- |
| 400 | Некорректные данные               |
| 403 | Пользователь не является учителем |

---

## 4. Изменить занятие

```http
PUT /activities/{activityId}
```

### Request body

```json
{
  "title": "Консультация по ОГЭ",
  "description": "Разбор письменной части ОГЭ",
  "subjectId": "uuid",
  "classId": "uuid",
  "date": "2026-09-16",
  "startTime": "15:30",
  "endTime": "16:15",
  "participantLimit": 15
}
```

### Успешный ответ `200 OK`

```json
{
  "id": "b7f3f6b2-2f43-4d8a-9c8f-9f2d1c1a1111",
  "status": "published"
}
```

### Ошибки

| Код | Когда                                            |
| --- | ------------------------------------------------ |
| 400 | Некорректные данные                              |
| 403 | Пользователь не может редактировать это занятие  |
| 404 | Занятие не найдено                               |
| 409 | Новый лимит меньше числа уже записанных учеников |

---

## 5. Отменить занятие

```http
PATCH /activities/{activityId}/cancel
```

### Request body

```json
{
  "reason": "Учитель заболел"
}
```

### Успешный ответ `200 OK`

```json
{
  "id": "b7f3f6b2-2f43-4d8a-9c8f-9f2d1c1a1111",
  "status": "cancelled"
}
```

### Ошибки

| Код | Когда                                      |
| --- | ------------------------------------------ |
| 403 | Пользователь не может отменить это занятие |
| 404 | Занятие не найдено                         |
| 409 | Занятие уже отменено или завершено         |

---

## 6. Записаться на занятие

```http
POST /activities/{activityId}/enrollments
```

### Успешный ответ `201 Created`

```json
{
  "id": "e2c4bcd7-72c4-4b91-8c0c-b1e6c6a22222",
  "activityId": "b7f3f6b2-2f43-4d8a-9c8f-9f2d1c1a1111",
  "status": "active",
  "enrolledAt": "2026-09-12T18:20:00"
}
```

### Ошибки

| Код | Когда                                                      |
| --- | ---------------------------------------------------------- |
| 403 | Пользователь не является учеником                          |
| 404 | Занятие не найдено                                         |
| 409 | Нет свободных мест / ученик уже записан / занятие отменено |

# Единый формат ошибки

Для всех ошибок используем один формат:

```json
{
  "code": "ACTIVITY_NOT_FOUND",
  "message": "Занятие не найдено"
}
```

Примеры кодов:

| Код ошибки           | Значение           |
| -------------------- | ------------------ |
| ACTIVITY_NOT_FOUND   | Занятие не найдено |
| ENROLLMENT_NOT_FOUND | Запись не найдена  |
| NO_AVAILABLE_PLACES  | Нет свободных мест |
| ALREADY_ENROLLED     | Ученик уже записан |
| ACTIVITY_CANCELLED   | Занятие отменено   |
| ACCESS_DENIED        | Недостаточно прав  |
| VALIDATION_ERROR     | Ошибка валидации   |

---

**Почему DELETE /enrollments/{id} не удаляет запись физически?**

Потому что для системы важна история действий. Поэтому операция меняет статус записи на `cancelled`, а не удаляет строку из БД.
